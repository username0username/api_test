Программные интерфейсы приложений и библиотек (API). Концепции построения. Интеграция в приложение. Надстройки и расширения ПО (addons and plugins).

Что такое API?

"Application Programming Interface", что дословно переводится как "интерфейс прикладного программирования" - это посредник, который позволяет двум программа обмениваться данными. Для простоты понимания я прикреплю схему (рисунок 1). То-есть, клиент создает запрос, условно GET, и получает с сервера необходимую информацию. Далее взаимодействие клиента с этой информацией может быть различным, например, вывод на экран.

Зачем нужен API?

Сейчас я приведу простой пример как со стороны пользователя, так и со стороны разработчика. - Пользователь имеет перед собой визуальный интерфейс приложения, в котором имеется кнопка "Конвертер валют". По нажатию на неё открывается ранее упомянутый конвертер, который получается с сервера. Пользователь не видит процесса обращения к серверу, он видит только интерфейс. - Разработчик может сэкономить время, подключив API стороннего сервиса к своему приложению, так как на сервере лежит уже готовая программа. Также, разработчику не обязательно знать внутренюю реализацию. О том, как происходит интеграция расскажу в следующих пунктах.

Преимущества API:

- Экономия времени разработчика - Множество сервисов, предоставляющих готовый продукт - Повторное использование в разных приложениях

Недостатки API:

- Не всегда обеспечивают необходимый функционал - Зависимость от других сервисов

Концепции построения:

- RESTful API использует стандартные операции HTTP чтобы обмениваться между клиентом и сервером. Идентефикация происходит через ЮРЛ, а данные передаются в форматах JSON или XML. Неплохо справляется с небольшими потоками данных. - SOAP API использует XML-сообщения чтобы обмениваться между компонентами программы. А здесь наоборот, настроен на больший поток данных - WebSocket API хорошо подойдет для постоянного коннекта между клиентом и сервером, что позволит передавать данные мгновенно. Обычно используют в чатах. - gRPC API использует HTTP/2 и Protocol Buffers для передачи данных. Передача данных потоковая, также подойдет для двухнаправленной передачи. Зачем нужен? Разные микросервисы. - GraphQL если вкратце, то только необходимые запросы, что делается для оптимизации.

Интеграция в приложения:

1. Пользовательская интеграция. Подключение происходит посредством специально разработанного апи. Вариант трудозатратный, но обеспечивает гибкость и контроль над интеграцией. 2. Коннекторы. Этот способ менее гибкий, но для подключения апи уже существует ПО от других разработчиков. 3. Интеграционные платформы. Они мощнее и универсальнее коннекторов, но значительно дороже. Включают в себя уже готовые коннекторы, могут выполнять сложные бизнес-процессы и многие другие возможности для людей, которые готовы много платить.

Надстройки и расширения ПО:

Аддоны - дополнения, расширяющие функциональность. Например, браузерные расширения. Плагины - тоже расширяют функциональность, но это уже программные компоненты, которые добавляются к приложению.
